3. How to read the drivers¶

Permutation importance shows how much ROC AUC drops when one transformed feature is shuffled on holdout data. Larger positive values usually mean the model depends on that signal more strongly. One-hot encoded columns appear as cat__<original>_<level>, so one original field can appear across multiple bars.

4. Product actions¶

First, prioritize retention tests for high-risk contract and payment groups that appear in both model drivers and segment charts. Second, strengthen onboarding for early-tenure customers and provide clearer support and security messaging where service features rank highly. Third, use income quartile as context for messaging and offer design, while keeping compliance checks in place.

Conclusion¶

This project asked whether Census income context improves churn prediction beyond internal telecom features. The pipeline cleaned leakage-prone fields, merged ACS income by state, engineered features, and compared Model A and Model B on the same stratified split.

On this run, logistic regression improved slightly with income features, while hist gradient boosting produced the same headline ROC AUC and PR AUC for A and B. Driver analysis still highlighted price sensitivity, month-to-month contracts, dependents, fiber service, and electronic check as key signals.

Use segment charts and model scores together to target outreach. Practical next steps include retention tests for high-risk contract and payment groups, stronger early-tenure onboarding, and refined affordability-focused messaging. If stronger lift from external data is required, test finer geographic granularity in a follow-up iteration.

Full write-up is in outputs/reports/conclusion.md. Resume-ready lines are in outputs/reports/resume_bullets.md.
